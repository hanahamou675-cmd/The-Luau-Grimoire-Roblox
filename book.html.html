1. The Vessel (Variables)

Variables store data. We use local so the script runs efficiently.

Pro-Tip: Use camelCase for names (like myCoins). Never start a name with a number!
local coins = 50
local name = "Player"

2. Forms of Matter

Luau needs to know if you are using text (Strings), math (Numbers), or logic (Booleans).

Pro-Tip: Booleans are only true or false. They are like light switches.
local speed = 16 -- Number
local isVip = true -- Boolean

3. Arithmetic

Math follows standard rules: + (Add), - (Sub), * (Mult), / (Div).

Pro-Tip: Use ^ for powers, like 2^3 for 8.
local health = 100 - 20
local double = health * 2

4. Decisions (If)

Control flow allows the script to branch. Use then and end.

Pro-Tip: == compares things. = sets a value. Don't mix them up!
if health < 50 then
    print("Warning!")
end

5. Cycles (Loops)

A "for" loop counts, while a "while" loop runs until something changes.

Pro-Tip: If your while loop has no task.wait(), the game will crash!
for i = 1, 5 do
    print("Loop: " .. i)
end

6. Rituals (Functions)

Functions store code for later use. Define once, use many times.

Pro-Tip: Use functions to avoid typing the same code twice.
local function sayHi()
    print("Hello!")
end

7. Exchange (Returns)

Functions can give data back to you using the return keyword.

Pro-Tip: Return acts like a "stop" sign‚Äînothing after it runs.
local function add(a, b)
    return a + b
end

8. Seeking (Wait)

Use WaitForChild for things that might take time to load.

Pro-Tip: This is essential for finding the Player's Head or Backpack.
local head = char:WaitForChild("Head")

9. Leaderstats

Create a folder named "leaderstats" in the player to show scores.

Pro-Tip: Case sensitive! It must be lowercase leaderstats.
local folder = Instance.new("Folder")
folder.Name = "leaderstats"

10. Scoping

Scope defines where a variable lives. Local variables stay inside their block.

Pro-Tip: Always use local to prevent "leaks" in your code.
do
    local secret = 5
end
-- secret is now nil
